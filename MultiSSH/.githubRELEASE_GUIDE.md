# Release Guide

This document explains how to create releases for MultiSSH using GitHub Actions.

## Quick Start

### Creating an Unsigned Release

1. Go to the **Actions** tab in your GitHub repository
2. Select the **"Build and Release"** workflow from the left sidebar
3. Click **"Run workflow"** button (top right)
4. Enter the version number (e.g., `1.0.0`)
5. Optionally check "Mark as pre-release" for beta versions
6. Click **"Run workflow"**

The workflow will:
- Build the app for macOS
- Create a DMG and ZIP file
- Generate SHA-256 checksums
- Create a GitHub release with all files attached

### Creating a Signed Release

For production releases with Apple Developer certificates:

1. First, configure your secrets (see below)
2. Go to **Actions** → **"Build and Release (Code Signed)"**
3. Follow the same steps as above

## Configuring Code Signing (Optional)

To create signed and notarized releases, you need to set up GitHub Secrets:

### Required Secrets

Go to **Settings** → **Secrets and variables** → **Actions** → **New repository secret**

| Secret Name | Description | How to Get It |
|------------|-------------|---------------|
| `CERTIFICATES_P12` | Base64-encoded .p12 certificate | See "Exporting Certificates" below |
| `CERTIFICATES_P12_PASSWORD` | Password for the .p12 file | The password you set when exporting |
| `CODE_SIGN_IDENTITY` | Signing identity name | Usually `"Developer ID Application: Your Name (TEAM_ID)"` |
| `DEVELOPMENT_TEAM` | Apple Team ID | Found in Apple Developer Portal |

### Optional Secrets (for Notarization)

| Secret Name | Description |
|------------|-------------|
| `NOTARIZATION_USERNAME` | Apple ID email address |
| `NOTARIZATION_PASSWORD` | App-specific password (not your Apple ID password) |

### Exporting Certificates

1. Open **Keychain Access** on your Mac
2. Select **My Certificates** in the left sidebar
3. Find your "Developer ID Application" certificate
4. Right-click and select **Export**
5. Save as `.p12` file with a strong password
6. Convert to base64:
   ```bash
   base64 -i certificate.p12 | pbcopy
   ```
7. Paste the result into the `CERTIFICATES_P12` secret

### Creating an App-Specific Password

1. Go to [appleid.apple.com](https://appleid.apple.com)
2. Sign in with your Apple ID
3. Go to **Security** section
4. Under **App-Specific Passwords**, click **Generate Password**
5. Enter a label (e.g., "GitHub Actions")
6. Copy the generated password and save it as `NOTARIZATION_PASSWORD`

## Workflow Details

### Unsigned Release Workflow

**File:** `.github/workflows/release.yml`

**Features:**
- ✅ Fast build times
- ✅ No Apple Developer account needed
- ✅ Good for testing and internal distribution
- ⚠️ Users need to bypass Gatekeeper on first launch

**Use Cases:**
- Development builds
- Internal testing
- Pre-release versions
- Quick distribution

### Signed Release Workflow

**File:** `.github/workflows/release-signed.yml`

**Features:**
- ✅ Code signed with Apple Developer certificate
- ✅ Notarized by Apple (if configured)
- ✅ No Gatekeeper warnings for users
- ✅ Professional distribution
- ⚠️ Requires Apple Developer account ($99/year)

**Use Cases:**
- Production releases
- Public distribution
- App Store-quality builds

## Version Numbering

Follow [Semantic Versioning](https://semver.org/):

- **Major version** (1.0.0 → 2.0.0): Breaking changes
- **Minor version** (1.0.0 → 1.1.0): New features, backwards compatible
- **Patch version** (1.0.0 → 1.0.1): Bug fixes

Examples:
- `1.0.0` - First stable release
- `1.1.0` - Added new features
- `1.1.1` - Bug fixes
- `2.0.0-beta.1` - Beta version (use pre-release checkbox)

## After Release

After the workflow completes:

1. Check the **Releases** page in your repository
2. Edit the release notes if needed
3. Announce the release to your users
4. Monitor for issues

## Troubleshooting

### Build Fails

**Check the Xcode version:**
```yaml
xcode-version: latest-stable  # or specific version like "15.2"
```

**Verify scheme name:**
The workflow uses `-scheme MultiSSH`. If your scheme name is different, update it in the workflow file.

### Code Signing Fails

**Verify secrets are set:**
- All secrets are correctly named (case-sensitive)
- Certificate password is correct
- Team ID matches your Apple Developer account

**Check certificate validity:**
```bash
security find-identity -v -p codesigning
```

### Notarization Fails

**Common issues:**
- App-specific password expired or incorrect
- Team ID doesn't match the certificate
- Binary has issues (unsigned frameworks, etc.)

**Check notarization logs:**
```bash
xcrun notarytool log <submission-id> \
  --apple-id "your@email.com" \
  --password "app-specific-password" \
  --team-id "TEAM_ID"
```

### DMG Creation Fails

**Disk space issues:**
The runner may run out of space. Clean up DerivedData:
```bash
rm -rf ./DerivedData/Build
```

## Manual Local Build

To build locally (matches the workflow):

```bash
# Unsigned build
xcodebuild clean build \
  -scheme MultiSSH \
  -configuration Release \
  -derivedDataPath ./DerivedData \
  -destination 'platform=macOS' \
  CODE_SIGN_IDENTITY="" \
  CODE_SIGNING_REQUIRED=NO

# Find the app
find ./DerivedData -name "MultiSSH.app" -type d

# Create DMG
hdiutil create -volname "MultiSSH" \
  -srcfolder path/to/MultiSSH.app \
  -ov -format UDZO \
  MultiSSH.dmg
```
